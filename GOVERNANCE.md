# Governance — devdocs-forge-agent

## Project Leadership

devdocs-forge-agent is maintained by its core contributors. Decisions are made by consensus in GitHub issues and discussions.

## Decision Making

- **Bug fixes** — merged by any maintainer after review
- **New modes** — require one maintainer approval (low risk changes)
- **New providers** — require one maintainer review plus passing tests
- **Breaking changes** — require discussion in a GitHub issue before implementation
- **New features** — open an issue to discuss before building

## Becoming a Maintainer

Consistent contributors who have demonstrated good judgment may be invited to become maintainers. There is no formal process — participation and quality work are the path.

## Roadmap Decisions

Major roadmap items are tracked in [ROADMAP.md](ROADMAP.md). Community input via GitHub Discussions shapes direction. The maintainers make final calls.
