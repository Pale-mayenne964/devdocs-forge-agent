# Mode: Blog Post

Generate a developer blog post from the provided transcript.

## Output Structure

1. **Title** — Compelling H1 that includes the primary keyword and topic
2. **Introduction** — 2-3 sentences that hook the reader and state what they'll learn
3. **Background / Context** — Why this topic matters (1-2 paragraphs)
4. **Main Content** — Break into logical H2 sections covering the key points from the transcript
5. **Code Examples** — Include relevant code snippets with explanations
6. **Key Takeaways** — 3-5 bullet points summarizing the most important lessons
7. **Conclusion** — 1-2 sentences wrapping up with a call to action
8. **Source Attribution** — Credit the original source if URL was provided
9. **Tags** — Suggest 3-5 relevant tags

## Writing Guidelines for Blog Format

- Write in first or second person ("you can", "here's how")
- Keep paragraphs short — 3-4 sentences maximum
- Use section headers to improve scannability
- Include at least one code example if present in the transcript
- Write the intro to work as a social media preview (first 2 sentences matter most)
- Include a clear value proposition: what the reader gains from reading

## SEO Considerations

- Place the primary keyword in the title and first paragraph
- Use descriptive subheadings that contain secondary keywords
- Aim for 600-1500 words for SEO effectiveness
- Include a meta description suggestion at the end
