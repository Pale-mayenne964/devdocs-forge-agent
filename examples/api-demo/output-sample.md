# Building a Simple AI Agent with Node.js

The simplest possible AI agent is a loop: receive a goal, pick a tool, run it, check the result, loop until done. This guide builds that loop in plain Node.js with no frameworks — so you can see exactly what is happening.

## Prerequisites

- Node.js 18 or later (native `fetch`, top-level `await`)
- An Anthropic API key (or adapt for OpenAI / Gemini)
- Basic JavaScript / ESM familiarity

## What You Will Build

An agent that:

1. Receives a natural language question
2. Checks a local knowledge base first
3. Falls back to the LLM if no match is found
4. Returns a final answer

This is the "tools first, model as fallback" pattern used in production agent systems.

## Project Setup

```bash
mkdir my-agent && cd my-agent
npm init -y
npm install @anthropic-ai/sdk dotenv
```

Create `.env`:

```
ANTHROPIC_API_KEY=your-key-here
```

Add `"type": "module"` to `package.json` to use ESM.

## Step 1: Bare LLM Call

```javascript
// agent.mjs
import Anthropic from '@anthropic-ai/sdk';
import 'dotenv/config';

const client = new Anthropic();

async function askModel(question) {
  const response = await client.messages.create({
    model: 'claude-3-5-sonnet-latest',
    max_tokens: 1024,
    messages: [{ role: 'user', content: question }],
  });
  return response.content[0].text;
}
```

## Step 2: Add a Tool (Knowledge Base Lookup)

```javascript
const knowledgeBase = {
  'node version': 'The current LTS version of Node.js is 20.x',
  'typescript release': 'TypeScript 5.5 was released in June 2024',
};

function searchKnowledge(query) {
  const match = Object.keys(knowledgeBase).find(k =>
    query.toLowerCase().includes(k)
  );
  return match ? knowledgeBase[match] : null;
}
```

## Step 3: The Agent Loop

```javascript
async function agent(question) {
  console.log(`[Agent] Question: ${question}`);

  // Tool check first
  const toolResult = searchKnowledge(question);
  if (toolResult) {
    console.log('[Tool] Knowledge base hit');
    return toolResult;
  }

  // Model fallback
  console.log('[LLM] No tool match — asking model');
  return await askModel(question);
}

// Run it
const answer = await agent('What is the current Node version?');
console.log(`[Answer] ${answer}`);
```

## How to Run

```bash
node agent.mjs
```

Expected output for a knowledge base hit:

```
[Agent] Question: What is the current Node version?
[Tool] Knowledge base hit
[Answer] The current LTS version of Node.js is 20.x
```

## Extending the Agent

| Extension | How |
|---|---|
| More tools | Add entries to `knowledgeBase` or integrate a real search API |
| Model tool selection | Use Anthropic's tool_use API to let the model choose tools |
| Multi-step reasoning | Wrap the agent loop so it can call tools multiple times |
| Memory | Append previous messages to the `messages` array |

## Key Takeaways

- An agent is a loop: goal → tool check → model fallback → answer
- "Tools first, model as fallback" reduces unnecessary API calls
- Production agents add tool selection by the model, retries, and memory
- The Anthropic SDK's `tool_use` feature handles structured tool calls natively
