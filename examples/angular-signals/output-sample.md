# Angular Signals: Reactive State Without the Complexity

Angular 17 introduced Signals as a stable reactive primitive — a simpler way to manage local component state and derived values without the subscription lifecycle overhead of RxJS.

## Prerequisites

- Angular 17 or later
- Basic familiarity with Angular components and templates
- Node.js 18+

## What Is a Signal?

A Signal is a reactive value container. When its value changes, Angular automatically schedules updates to any template or computed value that reads it — without manual `markForCheck()` calls or subscription teardown.

```typescript
import { signal } from '@angular/core';

const count = signal(0);
```

## Reading and Writing a Signal

Read a signal by calling it as a function:

```typescript
console.log(count()); // 0
```

Update it with `set()` (replace) or `update()` (derive from current value):

```typescript
count.set(5);
count.update(prev => prev + 1); // now 6
```

## Computed Signals

`computed()` creates a memoized derived value. It only recalculates when one of its signal dependencies changes:

```typescript
import { computed } from '@angular/core';

const doubleCount = computed(() => count() * 2);

console.log(doubleCount()); // 12 (after count = 6)
```

## Using Signals in a Component

Angular's template compiler automatically tracks signal reads and re-renders only the affected view nodes:

```typescript
import { Component, signal } from '@angular/core';

@Component({
  selector: 'app-counter',
  standalone: true,
  template: `
    <p>Count: {{ count() }}</p>
    <button (click)="increment()">+1</button>
  `,
})
export class CounterComponent {
  count = signal(0);

  increment() {
    this.count.update(v => v + 1);
  }
}
```

No `ChangeDetectionStrategy.OnPush` required. No `async` pipe. Angular tracks the signal read in `{{ count() }}` and updates only that binding.

## Signals vs RxJS: When to Use Which

| Scenario | Recommended |
|---|---|
| Local component state | Signal |
| Derived UI values | `computed()` |
| Simple shared state | Signal in a service |
| HTTP requests | RxJS (`HttpClient`) |
| Event streams with operators | RxJS |
| Timer / interval streams | RxJS |

### Interoperability

Convert an Observable to a Signal with `toSignal()`:

```typescript
import { toSignal } from '@angular/core/rxjs-interop';
import { inject } from '@angular/core';
import { HttpClient } from '@angular/common/http';

export class UserService {
  private http = inject(HttpClient);
  users = toSignal(this.http.get<User[]>('/api/users'), { initialValue: [] });
}
```

## Summary

- `signal(value)` — creates a readable/writable reactive value
- `computed(() => ...)` — creates a memoized derived value
- `signal.set(v)` / `signal.update(fn)` — updates the signal
- Angular templates track signal reads automatically
- Use Signals for state; keep RxJS for async streams
