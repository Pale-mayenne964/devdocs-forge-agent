# Frequently Asked Questions: Angular Signals

## Setup

### Do I need to install a separate package for Signals?

No. Signals are built into `@angular/core` starting from Angular 17. Just import `signal` and `computed` directly from `@angular/core`.

### Which version of Angular introduced Signals as stable?

Angular 17 (released November 2023) marked Signals as stable for production use. Earlier versions had an experimental implementation.

## Usage

### How do I read a Signal's value?

Call the signal like a function: `count()`. This is intentional — it allows Angular to track which templates and computed values depend on that Signal.

### What's the difference between `set()` and `update()`?

- `set(newValue)` — replaces the current value regardless of what it was
- `update(fn)` — receives the previous value and returns the new one

Use `update()` when the new value depends on the old one (e.g., incrementing a counter).

### Can I use Signals outside of Angular components?

Yes. Signals work in services, directives, and plain TypeScript classes. Place shared Signals in an injectable service for cross-component state:

```typescript
@Injectable({ providedIn: 'root' })
export class CartService {
  items = signal<string[]>([]);
}
```

### What is `computed()` and when should I use it?

`computed()` creates a derived Signal whose value is recalculated whenever its dependencies change. It is memoized — if dependencies haven't changed, it returns the cached value. Use it for any value that is derived from one or more other Signals.

## Signals vs RxJS

### Should I replace all my BehaviorSubjects with Signals?

Not necessarily. Signals are ideal for synchronous local state and derived UI values. BehaviorSubject and RxJS are still better for async streams, HTTP, debouncing, and complex event orchestration. Migrate incrementally where it makes sense.

### How do I use a Signal with an Observable?

Use `toSignal()` from `@angular/core/rxjs-interop` to convert an Observable to a Signal:

```typescript
import { toSignal } from '@angular/core/rxjs-interop';

const data = toSignal(this.http.get('/api/items'), { initialValue: [] });
```

### Do Signals work with OnPush change detection?

Yes, and they are even more efficient with `OnPush`. Angular tracks Signal reads in the template and only updates the component when those specific Signals change.

## Troubleshooting

### My template is not updating when the Signal changes.

Make sure you are calling the Signal in the template: `{{ count() }}` not `{{ count }}`. Without the parentheses, Angular does not register the dependency and won't update the template.

### `computed()` is returning stale data.

Verify that your `computed()` reads the Signal by calling it: `computed(() => count())` not `computed(() => count)`. The function argument must read the Signal directly for the dependency to be tracked.
