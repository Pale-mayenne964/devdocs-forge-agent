Angular 17 introduced Signals as a stable reactive primitive. If you've been managing state with RxJS Subjects and `async` pipes, Signals offer a simpler path for local component state and derived values.

## Prerequisites

- Angular 17 or later installed
- Basic familiarity with Angular components and TypeScript
- Node.js 18+ and npm

## What Is a Signal?

A Signal is a reactive value container. When its value changes, Angular automatically identifies which parts of the template depend on it and schedules only those updates — no manual `markForCheck()`, no subscription teardown.

```typescript
import { signal } from '@angular/core';

const count = signal(0);
```

## Reading and Writing Signals

**Read** a Signal by calling it like a function:

```typescript
console.log(count()); // 0
```

**Write** using `set()` for a new value, or `update()` to derive from the previous:

```typescript
count.set(5);
count.update(prev => prev + 1); // now 6
```

## Computed Signals

`computed()` creates a derived Signal that recalculates only when its dependencies change:

```typescript
import { computed } from '@angular/core';

const doubleCount = computed(() => count() * 2);
```

`doubleCount` is memoized — if `count` hasn't changed, reading `doubleCount` returns the cached value.

## Using Signals in a Component

```typescript title="counter.component.ts"
import { Component, signal, computed } from '@angular/core';

@Component({
  selector: 'app-counter',
  template: `
    <p>Count: {{ count() }}</p>
    <p>Double: {{ doubleCount() }}</p>
    <button (click)="increment()">+1</button>
  `,
})
export class CounterComponent {
  count = signal(0);
  doubleCount = computed(() => this.count() * 2);

  increment() {
    this.count.update(v => v + 1);
  }
}
```

:::tip
Angular's template compiler detects Signal reads automatically. No zone.js patching or `OnPush` configuration required for Signals to work efficiently.
:::

## Signals vs RxJS: When to Use Which

| Scenario | Recommended |
|----------|------------|
| Local component state | Signal |
| Derived UI values | `computed()` Signal |
| HTTP requests | RxJS + `HttpClient` |
| Event streams (debounce, etc.) | RxJS |
| Cross-component shared state | Signal (with service) |
| Complex async orchestration | RxJS |

To bridge the two systems, use `toSignal()` from `@angular/core/rxjs-interop`:

```typescript
import { toSignal } from '@angular/core/rxjs-interop';
import { interval } from 'rxjs';

const tick = toSignal(interval(1000), { initialValue: 0 });
```

## Summary

- `signal(value)` creates a reactive container
- `signal.set()` and `signal.update()` write new values
- `computed()` creates memoized derived Signals
- Templates read Signals by calling them: `{{ count() }}`
- Use Signals for local state; RxJS for async streams
